## Работа с ОС

Зачем важно знать о взаимодействии с ОС?
1) Знание особенностей ОС позволяет лучше предотвращзать сбои и уязвимости
2) Понимание взаимодействия с ОС позволяет лучше понимать, как работает язык и его библиотеки
3) Понимание, как Golang взаимодействует с системой, помогает оптимизировать использование CPU и памяти

Параметры взаимодействия с ОС в Golang
1) Пакеты os, syscall (запись, чтение файлов, работа с каталогами etc...)
2) Сетевые вызовы
3) Выделение ресурсов (памяти, процессорного времени)

Абстаркция от деталей ОС
1) Большая часть вашего кода может оставаться неизменной независимой от того, на какой ОС он запускается
2) Golang скрывает сложности взаимодействия с разными ОС, представляя единый интерфейс
3) Это делает язык иделаьным выбором для создания приложений, которые должны работать на различных опреационных системах, будь то Linux, Windows или macOS

## Что такое планировщик

Зачем нужен планировщик:
1) Распределять вычислительные ресурсы между задачами
2) У ОС есть свой собтвенный планировщик
3) Почему бы не запускать по одной горутине на отдельном потоке ОС

Проблемы потоков:
1) Потоки дорогие по памяти (из-за стэка)
2) Потоки дорого преключать (сисколы, контекст)
Решения в Go:
1) Go использует growable stack
2) Go выбирает моменты, когда переключение дешевое
