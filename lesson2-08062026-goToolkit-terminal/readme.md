## Компиляция
Сборка программы, включающая:
Трансляцию всех модулей программы, написанных на одном или нескольких исходных языках программирования, в эквивалентные программные модули на низкоуровневом языке, близком машинному коду;
Поседующую сборку (линковку) исполняемой машинной программы, в том числе вставку в программу кода всех функций, импортируемых из статических библиотек.

Интересный факт: компилятор Go написан на самом Go (с версии 1.5)

Этапы компиляции:
1) Лексически и синтаксический разрбор - проверка на текстовые ошибки.
2) Построение АСД - абстрактное синтаксическое дерево. Проверка на типы
3) Подготовка промежуточного представления - подготовка к оптимизациям.
4) Упрощение синтаксиса - трансформация сложных конструкций языка в более простые.
5) Генерация SSA (незкоуровневое представление) - удаление мертвого кода, оптимизация потоков.
6) Генерация машинного кода - перевод в ASM
7) Линковка - объединение в один файл.
https://go.dev/src/cmd/compile/README

go run main.go (компиляция + запуск)
go build main.go (компиляция (создает исполняемый файл текущей архитектуры))
go build -o hello-program main.go (-o hello-program - название исполняемого файла)
go build -gcflags="-S" main.go 2> output.asm - вывод в assebler формате

GOOS=windows go build -o /path/to/program main.go
GOARCH=386 GOOS=darwin go build -o /path/to/program main.go
GOARCH=amd64 GOOS=linux go build -o test-linux main.go

## Пакеты и модули

### Модули
go get
go install
go test

Любой модуль начинается с файта go.mod
go mod init my_module
go get github.com/gin-gonic/gin - загружает библиотеки в текущий модуль
go mod tidy - оставить только нужное
go get github.com/gin-gonic/gin@v1.11.0 - определённая версия библиотеки

### Пакеты
package - пространство имён, как куски кода отделить друг от друга
Модуль можно вызвать из модуля.
Нельзя вызвать из двух модулей друг друга - получается цикличность
go get path/to/module/... - качает и модуль и все зависимости (вложенные модули)

### WorkSpace
Позволяет работать с различными модулями расположенными в одной директории проекта.

go work init - создать пространство
go work use - использовать модуль
go work replace - заменить модуль

### Подпрограммы

go vet - 
go tool pprof - как работает программа
go tool cover - как работает, какие пути у функций
go tool trace - как работает, какие пути у функций
go doc - чтение документации (если описана в коде)
go help tool - чтение справки по tool

## Организация проекта

/cmd - основные приложения для текущего проекта
/internal - внутренний код и библиотеки, которые не хотим никому показывать
/pkg - код библиотек, которыми хотим поделиться
/vendor - зависимости приложения
/config - конфигурационные файлы

### Best practice
Лучше много небольших файлов, чем один большой
Удобство навигации
Располагайте типы в начале файла
Организовывайте типы по их функциональному назначению
Используейте Godoc
Не пишите бизнес-логику в main.go

### Конвенции именования
lowercase слитный
Старайтесь избегать частоиспользуемых имён
Используйте единственное число


## Форматирование и линтинг

### Форматирование кода

В Go нет style guide, зато есть go fmt /path.to/code.go - применяет базовое форматирование кода

Это продвинутое форматирование

go get golang.org/x/tools/cmd/goimports

### Линтеры

go vet - встроенный линтер - программа, анализирующая код и сообщающая о потенциальных проблемах.
go vet ./... - проверить все файлы
golint - популярный сторонний линтер
